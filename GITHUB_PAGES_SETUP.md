# GitHub Pages Deployment Guide

## Overview
The Meh-Scanner dashboard automatically deploys to GitHub Pages at:
```
https://<username>.github.io/<repo-name>/
```

## Deployment Architecture

### Key Files Generated
- **public/index.html** — Interactive dashboard (generated by `dashboard_export.py`)
- **public/latest.json** — JSON API with today's deals (generated by `dashboard_export.py`)
- **public/.nojekyll** — Disables Jekyll processing (empty file, required!)

### Why .nojekyll?
GitHub Pages uses Jekyll by default, which can:
- Strip certain HTML/JS features
- Cause 404 errors on pages without extensions
- Interfere with `<base>` tag routing

Creating `.nojekyll` at the repository root tells GitHub Pages to serve files as-is, without Jekyll processing.

### Why <base> tag?
GitHub project sites (e.g., `user.github.io/repo-name/`) require a `<base href="/repo-name/">` tag in the HTML `<head>` so that relative links (`./latest.json`, `./style.css`) resolve correctly under the project path.

**Exception:** If your repo is named `<username>.github.io` (user site), it uses the root `/` and doesn't need the base tag.

## First-Time Setup

### Step 1: Verify Local Tests
```bash
# Test without base path (local)
python main.py

# Test with base path (simulates GitHub Pages)
export MEH_DASHBOARD_BASE_PATH="/meh-scanner/"
python main.py

# Or run the full test suite
bash test_github_pages.sh
```

Expected output:
- ✅ `public/index.html` created (500+ bytes)
- ✅ `public/latest.json` created
- ✅ `public/.nojekyll` created (empty file)
- ✅ `<base href="/meh-scanner/">` injected when MEH_DASHBOARD_BASE_PATH is set

### Step 2: Push & Run Workflow
```bash
git add .
git commit -m "Add GitHub Pages deployment"
git push
```

The `daily-meh-scanner.yml` workflow will:
1. Run the scanner
2. Generate dashboard files in `public/`
3. Verify all files are present (.nojekyll, index.html, latest.json)
4. Deploy to the `gh-pages` branch

### Step 3: Configure GitHub Pages Settings (One-Time)
1. Go to: **[Repo Settings](https://github.com/settings/repositories)** → **Your Repo** → **Settings** tab
2. Scroll to **"Pages"** section
3. Under **"Build and deployment"**:
   - **Source:** Select `Deploy from a branch`
   - **Branch:** Select `gh-pages`
   - **Folder:** Select `/(root)`
4. Click **Save**

### Step 4: Wait & Verify
- GitHub Pages needs 1–2 minutes to initialize on first deploy
- Visit: `https://<username>.github.io/<repo-name>/`
- If you see a 404 or blank page:
  - **Hard-refresh** (Ctrl+Shift+R on Windows/Linux, Cmd+Shift+R on Mac)
  - Wait another 30 seconds
  - Check that Settings → Pages shows "Your site is live at..."

## Troubleshooting

### 404 or Blank Page (Most Common)
**Symptoms:**
- URL returns 404 or blank white/dark page
- Page loads but no table or content visible
- Assets (CSS/JS) load but page is broken

**Diagnosis (Browser DevTools):**
1. Open site: `https://<username>.github.io/<repo>/`
2. Press **F12** → **Network** tab
3. Hard-refresh (Ctrl+Shift+R)
4. Check for 404 errors:
   - If `latest.json` returns 404 → base path issue
   - If root URL returns 404 → Pages settings issue
   - If page loads but blank → check **Console** tab for JS errors

**Fix (Priority Order):**

**Step 1: Verify GitHub Pages Settings (One-time)**
1. Go to: **Repo Settings** → **Pages**
2. Look for: "Your site is live at: https://..."
   - If you see this ✅, move to Step 2
   - If not, configure:
     - Source: `Deploy from a branch`
     - Branch: `gh-pages`
     - Folder: `/(root)`
     - Click **Save**
   - Wait 1-2 minutes for Pages to initialize

**Step 2: Verify gh-pages Branch Has Files**
1. Go to: **Code** → Switch to `gh-pages` branch
2. You should see:
   - `index.html` (main file)
   - `latest.json` (data file)
   - `.nojekyll` (empty file, in root)
3. If `.nojekyll` is missing:
   - Check workflow logs: **"Check dashboard output"** → should show ✅
   - If ❌ there, dashboard generation failed
   - If ✅ but missing from gh-pages, re-run workflow

**Step 3: Hard-Refresh & Wait**
```
Ctrl+Shift+R (Windows/Linux) or Cmd+Shift+R (Mac)
```
- Wait 30 seconds
- Check if dashboard loads (dark table with deals)
- First deploy may take 1-2 minutes

**Step 4: Check Browser Cache**
If still blank:
1. **Chrome/Edge:** DevTools → Settings (⚙️) → Network → "Disable cache"
2. Hard-refresh again
3. Open DevTools Console (F12 → Console) for JS errors

### Base Path / Asset 404s
**Symptom:** Page loads but "latest.json not found" or table won't populate

**Cause:** Base path not injected or resolving incorrectly

**Diagnosis:**
1. Open site → DevTools → Network tab
2. Look for request to `latest.json` → what's the full URL?
   - Should be: `https://<username>.github.io/<repo>/latest.json`
   - If it's: `https://<username>.github.io/latest.json` → base path missing

**Fix:**
1. Check workflow "Verify HTML base tag" step in logs
   - Should show: `<base href="/repo-name/">`
   - If missing, dashboard export failed
2. Check local version:
   ```bash
   export MEH_DASHBOARD_BASE_PATH="/<repo-name>/"
   python main.py
   grep "base href" public/index.html
   ```
   Should show: `<base href="/repo-name/">`

### .nojekyll Missing (Jekyll Interference)
**Symptoms:**
- Page serves but content jumbled
- CSS works but styling broken
- Unexpected HTML transformations

**Diagnosis:**
1. Go to gh-pages branch on GitHub
2. Look for `.nojekyll` file at root (not in a subfolder)
3. Check workflow "Check dashboard output" → `.nojekyll` should be ✅

**Fix:**
1. Verify locally:
   ```bash
   ls -la public/.nojekyll
   ```
   Should exist and be empty (0 bytes)
2. Re-run workflow:
   - Go to **Actions** → **Daily Meh-Scanner** → **Run workflow** → **Run workflow**
   - Wait for completion
3. Check gh-pages branch again

### "Jekyll processing interfered" (Rare)
**Symptom:** Pages serves HTML but with Jekyll-injected content

**Cause:** `.nojekyll` not at repository root

**Fix:**
1. Manually commit `.nojekyll`:
   ```bash
   echo "" > .nojekyll
   git add .nojekyll
   git commit -m "Add .nojekyll to disable Jekyll"
   git push origin main
   ```
   Workflow will automatically deploy to gh-pages
2. Verify it appears on gh-pages branch

### Pages Settings Shows "Not Published"
**Symptom:** Settings → Pages shows red error or "Not configured"

**Cause:** Source not set correctly or gh-pages branch doesn't exist

**Fix:**
1. Go to **Settings** → **Pages**
2. Change **Source** to `Deploy from a branch`
3. Select **Branch:** `gh-pages`
4. Select **Folder:** `/ (root)`
5. Click **Save**
6. Wait 1-2 minutes
7. Should show: "Your site is live at: https://..."

## Manual Fallback (if workflow fails)

If the workflow doesn't deploy:

```bash
# 1. Generate dashboard locally
export MEH_DASHBOARD_BASE_PATH="/<repo-name>/"
python main.py

# 2. Verify files
ls -la public/
# Should show: index.html, latest.json, .nojekyll

# 3. Manually push to gh-pages
git add -f public/
git commit -m "Manual dashboard deploy"
git push origin gh-pages --force

# 4. Verify GitHub Pages settings (as above)
```

## Verification Checklist (After Pushing)

### Before Opening the Live URL
- [ ] Workflow has completed (check **Actions** tab)
- [ ] **Status** shows `Deploy: success` (green ✅)
- [ ] **Dashboard Files** table shows all three files present (✅)

### After Opening the Live URL
1. Open: `https://<username>.github.io/<repo>/`
2. You should see:
   - [ ] Dark-themed table with column headers (Score, Site, Niche, Price, etc.)
   - [ ] Filter search box at top
   - [ ] "Min score" dropdown filter
   - [ ] "Last updated" timestamp
   - [ ] Deal rows with data (if any deals qualified)
   - [ ] Color highlights: blue (score ≥7), green (score ≥8)
3. Test interactive features:
   - [ ] Type in filter box → table filters in real-time
   - [ ] Change min score → table updates
   - [ ] Click column headers → rows sort
   - [ ] Click deal links → opens site in new tab

### Checking DevTools for Issues
1. Press **F12** → **Network** tab
2. Hard-refresh (Ctrl+Shift+R)
3. Check **Status** column:
   - [ ] `index.html` = 200
   - [ ] `latest.json` = 200 (if filter working)
   - [ ] No red 404s
4. If any 404s:
   - Note the URL (e.g., `/meh-scanner/latest.json` vs `/latest.json`)
   - This indicates a base path issue
   - Review the **Base Path / Asset 404s** troubleshooting section above

### Post-Deploy Diagnostics
If things aren't working, collect this info:

**Workflow Summary Check:**
- Go to **Actions** → most recent run
- Confirm:
  - ✅ Dashboard Files table: all three files present
  - ✅ Deployment status: "success"
  - ✅ Base tag verification showed correct path

**Browser DevTools:**
```
F12 → Console tab:
- Any red error messages?
- Check for CORS errors or missing asset messages

F12 → Network tab:
- latest.json URL: should be https://<user>.github.io/<repo>/latest.json
- If missing, check the base tag in View Source
```

**gh-pages Branch Check:**
- Go to **Code** → select **gh-pages** branch
- Files present?
  - index.html ✅
  - latest.json ✅
  - .nojekyll ✅ (empty file)
